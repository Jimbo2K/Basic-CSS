Un padre es un elemento que contiene a otro directamente: BODY es padre de HEADER pero no de H1

Un hijo es un elemento contenido directamente en otro elemento: LI es hijo de UL pero no de SECTION

Un ancestro es un elemento que contiene, no necesariamente de manera directa, a otro: BODY y HEADER son ancestros de H1

Un descendiente es cualquier elemento contenido en otro aunque no sea de manera directa: todos los elementos del documento son descendientes de HTML

Se denominan hermanos o siblings a los elementos que descienden de un mismo padre: los elementos de igual color son hermanos, por ejemplo HEADER SECTION y FOOTER

Estos conceptos son importantes ya que algunas propiedades CSS son heredadas de padres a hijos, y de ancestros a descendientes. Y también los podemos utilizar mediante selectores para asignar reglas css.

Normalmente consideraremos como ancestro principal al <body> ya que los estilos css sólo tiene sentido aplicarlos a elementos que sean accesibles al usuario


Selector universal *

Si utilizamos * como selector en una regla css indicaremos que se aplique dicha regla a cualquier elemento del body.

Lo normal es utilizarlo en la primera regla css que utilicemos para realizar un reset o set inicial de propiedades básicas que se heredan

En el siguiente ejemplo se han creado tres elementos caja, cada uno hijo del anterior:

CSS:

.caja1 { border: 3px solid; } .caja2 { border: 2px solid; } .caja3 { border: 1px solid; }

Selector de etiquetas

Se puede utilizar como selector de reglas css el nombre de cualquier etiqueta html.

De esta a cualquier elemento HTML se le aplicará el estilo definido con el selector que lleve su nombre.

article { font-family: verdana, sans-serif; color: darkgrey; text-align: justify; } h1 { fontweight: bolder; color: blue; } p, a { font-family: verdana, sans-serif; color: darkgrey; } p { text-align: justify; }

En el ejmplo se vé como se puede repetir un selector varias veces en distintas reglas (en el caso de <p>). Esto se puede hacer para declarar otras nuevas propiedades (como en el ejemplo) porque si repetimos selector y propiedades, el navegador utilizará las últimas propiedades repetidas que hayamos declarado


Selector de clase

Este tipo de selector lo podemos utilizar para aplicar estilos a elementos HTML que tenga declarada una clase en sus atributos: <p class="nombre_clase">.

Para crear estos selectores debemos empezar escribiendo "." y seguido el nombre de la clase: .nombre_clase.

Podemos utilizar estos selectores de manera general, aplicando el estilo a cualquier elemento HTML que utilice la clase.

.nombre_clase { color: red; font-style: italic; }

También podemos concretar a que tipo de elementos concretos que utilicen la clase se les va a aplicar el estilo.

<p class="nombre_clase">Esto es un párrafo<p> <a href="#" class="nombre_clase">Enlace a ninguna parte<p> .nombre_clase { color: red; text-decoration: none; } p.nombre_clase { font-weight: bold; }

Esto es un párrafo

Enlace a ninguna parte

Se pueden asignar multiples clases a un mismo leemento HTML: <p class="parrafo color_rojo texto1">Parrafo con tres clases<p>


Selector de id

Este tipo de selector lo podemos utilizar para aplicar estilos a un único elemento HTML que tenga declarado un id en sus atributos: <header id="nombre_id">.

Para crear estos selectores debemos empezar escribiendo "#" y seguido el nombre del id: #nombre_id.

Al ser la declaración de un id en HTML única lo normal suele ser utilizarlo como selector único.

#nombre_id { color: darkgrey; font-family: arial, sans-serif; }

Aún así es posible que utilicemos un mismo id en distintos documentos HTML y para distintos elementos HTML a los que hay que aplicar distintos estilos

En un documento HTML de utiliza: <h1>Título de la WEB</h1> <p id="principal">Descripción de la página web</p> Y en otro documento: <article id="principal"> <h2>Último artículo publicado</h2> ... </article>
En lugar de crear dos hojas css: #principal { font-family: verdana; } p#principal { font-size: 1.1em; } article#principal { font-size: 1em; font-style: italic; }

En ambos documentos p y article utilizarán la misma tipografía (verdana), pero en p el texto será de mayor tamaño y en article de tamaño normal y estilo itálica


Selector descendente

Este selector se utiliza para aplicar estilos a elementos que son descendientes de otros.

Un selector descendente se compone de un selector que identifica al ancestro, selector que identifica al descendiente o descendientes, ambos separados por un espacio en blanco.

HTML
<p>Un ejemplo de <strong>selector descendente</strong></p> <section> <p>El siguiente es <strong>descendiente</strong> de section </section>
CSS
section strong { color: red; }

Un ejemplo de selector descendente

El siguiente texto es descendiente de section

La regla aplica texto de color rojo solo a las etiquetas strong que sean descendientes de una etiqueta section.


Una variable del selector descendente consiste en utilizar el selector universal * entre el selector ancestro y descendiente.

Como en el caso anterior la regla se aplicará a cualquier descendiente con el selector indicado menos a los descendientes directos, es decir, menos a los que sean hijos.


Caja1 * Caja { border-color: blue; }

A cualquier descendiente de Caja 1 menos a sus hijos se le aplica un color de borde azul

Selector de hijos

Estos selectores se utilizan para aplicar estilos a elementos que son hijos de otros.

Se forman con un selector que identifica al elemento padre, otro selector que identifica al hijo/s, ambos separados por ">".

HTML
<section> <h2>Encabezado de sección</h2> <p>Un párrafo cualquiera</p> <a href="#">Un enlace a ningún sitio</a> <p>Otro párrafo cualquiera</p> </section>
CSS
section > p { color: red; font-style: italic; }

Encabezado de sección

Un párrafo cualquiera

Un enlace a ningún sitio

Otro párrafo cualquiera

Si en lugar de utilizar el selector de elemento hijo utilizamos *, la regla se aplicará a cualquier elemento que sea hijo del indicado por el selector padre.

HTML
<section id="hijo"> <h2>Encabezado de sección</h2> <p>Un párrafo cualquiera</p> <a href="#">Un enlace a ningún sitio</a> <p>Otro párrafo cualquiera</p> </section>
CSS
#hijo > * { color: blue; font-style: italic; }

Encabezado de sección

Un párrafo cualquiera

Un enlace a ningún sitio

Selector de hermano

Este selector se compone de un selector hermano 1, otro hermano 2, ambos separados por el símbolo +.

La regla que utiliza este selector aplica el estilo al siguiente elemento que sea hermano de hermano1 y del tipo indicado por hermano2.

HTML
<p>Un párrafo</p> <a href="#">Enlace a ningún sitio</a> <div>Un bonito div</div> <p>Otro párrafo</p>
CSS
p + a { color: red; }
a + p { color: blue; }

La primera regla aplica color rojo al primer hermano de un párrafo que sea un ancla.

La segunda aplica azul al primer hermano de un ancla que sea un párrafo. Lo que pasa es que el primer hermano de <a> es un div y el siguiente párrafo no es hermano directo por lo que no se aplica.

Selector de hermanos

Este selector se compone de un selector hermano 1, otro hermano 2, ambos separados por el símbolo ~.

La regla que utiliza este selector aplica el estilo a los siguientes elementos que sean hermanos de hermano1 y del tipo indicado por hermano2.

HTML
<p>Un párrafo</p> <a href="#">Enlace a ningún sitio</a> <div>Un bonito div</div> <p>Otro párrafo</p> <p>Y otro más</p>
CSS
p ~ a { color: red; }
a ~ p { color: blue; }

Sin embargo esta vez con la segunda regla si hemos logrado aplicar el color azul al <p> que sigue a <a> pese a no ser hermano directo. Además también se aplicado el estilo al resto de párrafos hermanos de <a>.

Selector de atributos

Este selector permite aplicar reglas de estilo a elementos en los que se haya declarado un atributo concreto, incluso especificando que dicho atributo tenga un valor concreto.

La sintáxis del selector puede variar en función de si se determina o no el valor del atributo:

  • tipo_elemento[nombre_atributo]: este selector apunta a cualquier elemento del "tipo_elemento" que contenga el atributo "nombre_atributo"
  • tipo_elemento[nombre_atributo=valor]: sólo los "tipo_elemento" cuyo "nombre_atributo" sea "valor"
  • tipo_elemento[nombre_atributo]~=valor]: sólo los "tipo_elemento" cuyo "nombre_atributo" contenga entre sus valores "valor"
  • tipo_elemento[nombre_atributo]|=valor]: sólo los "tipo_elemento" cuyo "nombre_atributo" contenga un conjunto de palabras separadas por guiones y que comiencen por "valor"

CSS 3 ha añadido además tres nuevos operadores para los selectores de atributos:

  • tipo_elemento[nombre_atributo^=valor]: sólo los "tipo_elemento" cuyo "nombre_atributo" tenga asignado un valor que comience exactamente por la cadena "valor"
  • tipo_elemento[nombre_atributo]$=valor]: sólo los "tipo_elemento" cuyo "nombre_atributo" tenga asignado un valor que termine exactamente con la cadena "valor"
  • tipo_elemento[nombre_atributo]*=valor]: sólo los "tipo_elemento" cuyo "nombre_atributo" tenga asignado un valor que contenga en algún punto la cadena "valor"
